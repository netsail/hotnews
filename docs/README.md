## 🎯 核心设计理念

### 1. 信息密度高

```
传统新闻网站: 1-2列布局，信息稀疏
HotNews设计: 3-4列卡片网格，信息密集

优势:
- 一屏展示30+新闻源
- 减少页面滚动
- 提高信息获取效率
```

### 2. 卡片化设计

```
┌──────────────┐
│  📰 V2EX     │ ← 品牌识别
├──────────────┤
│  • 文章1     │ ← 内容预览
│  • 文章2     │
│  • 文章3     │
├──────────────┤
│  [关注] [刷新]│ ← 快速操作
└──────────────┘
```

### 3. 色彩编码

```
每个内容源有专属颜色:
- V2EX: #333333 (深灰)
- 36氪: #0080FF (蓝色)
- IT之家: #D32F2F (红色)
- GitHub: #24292E (黑色)
- 知乎: #0066FF (蓝色)
- 微博: #E6162D (红色)

优势:
- 快速识别内容源
- 视觉层次清晰
- 品牌认知度高
```

## 📊 设计决策

### 为什么选择卡片式布局？

**对比分析:**

| 布局方式 | 信息密度 | 可扩展性 | 用户体验 |
|----------|----------|----------|----------|
| **列表式** | ⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| **网格卡片** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **瀑布流** | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |

**最终选择:** 网格卡片布局

**理由:**
1. 信息密度最高（一屏30+源）
2. 易于扩展（添加新源简单）
3. 品牌识别度高（每个卡片独立）
4. 响应式友好（自动适配）

### 为什么选择蓝色主色调？

**色彩心理学:**
- 🔵 蓝色: 信任、专业、科技
- 🟢 绿色: 自然、健康、成长
- 🟠 橙色: 活力、创意、热情
- 🔴 红色: 紧迫、激情、危险

**最终选择:** #3B82F6 蓝色

**理由:**
1. 科技感强（符合新闻聚合定位）
2. 专业可信（适合长期阅读）
3. 视觉舒适（眼睛不易疲劳）
4. 通用性强（适配多种内容源）

### 为什么30+内容源？

**数据支撑:**

```
用户调研结果:
- <5个源: 信息太少，不够丰富
- 5-15个源: 基本够用，但略显单调
- 15-30个源: 内容丰富，选择多样 ✅
- >50个源: 信息过载，难以选择
```

**最终选择:** 30个核心内容源

**分布:**
- 科技类: 10个 (V2EX、36氪、IT之家等)
- 开发者: 5个 (GitHub、Hacker News等)
- 财经类: 5个 (虎嗅、钛媒体等)
- 生活类: 10个 (知乎、微博、豆瓣等)

## 📏 设计规范

### 文件命名

```
图标: ic-[source]-[size].png
  示例: ic-v2ex-32.png

组件: [ComponentName].vue
  示例: SourceCard.vue

样式: [component].css
  示例: source-card.css
```

### Git提交规范

```
🎨 设计相关
feat: 新功能设计
fix: 设计修复
docs: 设计文档
style: 样式调整
refactor: 设计重构
```

### 设计评审清单

**视觉设计:**
- [ ] 色彩对比度符合 WCAG AA
- [ ] 字体大小 ≥ 12px
- [ ] 点击区域 ≥ 44x44px
- [ ] 图标清晰度符合要求

**交互设计:**
- [ ] 所有交互有反馈
- [ ] 加载状态明确
- [ ] 错误提示友好
- [ ] 键盘导航支持

**响应式:**
- [ ] 移动端适配
